## ✅ 已實作的改善

### 1. 加入 prefers-reduced-motion 支援

**檔案**: `app/globals.css`

**改動**:
```css
/* Respect user's motion preferences for accessibility */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

**影響**:
- ✅ 符合 WCAG 2.1 Level AA 無障礙標準
- ✅ 保護前庭障礙患者避免動畫引發的不適
- ✅ 尊重使用者的作業系統偏好設定

**如何測試**:
- macOS: 系統設定 → 輔助使用 → 顯示器 → 減少動態效果
- Windows: 設定 → 輕鬆存取 → 顯示 → 在 Windows 中顯示動畫
- 開發者工具: Chrome DevTools → Rendering → Emulate CSS media feature prefers-reduced-motion

### 2. 增大行動版 touch target

**檔案**: `components/layout/SiteHeader.tsx`

**改動**:

**桌面版導航** (第 40 行):
```tsx
// Before: px-2 py-1 (約 32-36px)
// After:  px-3 py-2 md:px-2 md:py-1 (行動版 44px+, 桌面版保持原樣)
className="rounded-full px-3 py-2 md:px-2 md:py-1 transition-colors..."
```

**行動版選單** (第 93 行):
```tsx
// Before: px-2 py-1 (約 32-36px)
// After:  px-3 py-2 (44px+)
className="block rounded-md px-3 py-2 transition-colors..."
```

**影響**:
- ✅ 符合 Apple HIG 建議的最小 44x44pt touch target
- ✅ 符合 Material Design 建議的最小 48x48dp touch target
- ✅ 減少行動裝置上的誤觸
- ✅ 改善單手操作體驗
- ⚠️ 桌面版保持原樣,不影響現有設計

**如何測試**:
- 在實體手機上測試導航點擊
- Chrome DevTools 切換到行動裝置模擬
- 確認按鈕易於點擊,不會誤觸相鄰項目
