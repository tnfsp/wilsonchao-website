---

## 核心問題

**要做的是「體驗」，不是「介面」。**

目前風險：打開 → 看數字 → 點按鈕 → 看結果 → 交卷。像表單，不像 ICU。
缺少：敘事弧線、情緒起伏、不確定性。

---

## 真實 ICU 被 Call 的七步體驗弧線

模擬器應重現這個弧線。每個模式按各自方式呈現。

| # | 階段 | 真實體驗 | 設計目標 |
|---|------|---------|---------|
| 1 | **電話響** | 不知多嚴重，心裡跑 differential | 護理師描述要模糊、不完整 |
| 2 | **走到床邊** | 第一眼：臉色、呼吸、monitor 波形 | 視覺衝擊 — 波形 + 病人外觀描述 |
| 3 | **資訊碎片化** | 護理師講一半被打斷、lab 沒回、echo 要借 | 資訊不一次給完，要主動獲取 |
| 4 | **壓力遞增** | 病人在變差，護理師在等你下 order | 時間壓力 + 護理師催促 |
| 5 | **做決定** | 不確定對不對，但必須做 | 決策有後果，vitals 會變 |
| 6 | **等待結果** | 盯著 monitor 看有沒有改善 | 開藥後要等，看 vitals 漸變 |
| 7 | **收尾或崩盤** | 穩住交班 / 惡化叫人 | 交班 + escalation 路徑 |

## 六大設計原則

### 原則 1：遊戲中不教學

> **教學是完成之後的事情。**

- 遊戲中只給**真實會有的資訊**：護理師回報、monitor 變化、lab 結果
- ❌ 不要在 order 旁顯示「為什麼」
- ❌ 不要即時告訴玩家做對或做錯
- ❌ 不要 inline guideline feedback（即使 Standard 模式）
- ✅ **後果本身就是教學** — 開錯藥 → vitals 惡化
- ✅ 所有教學放在 **Debrief**
- ✅ 護理師的「暗示」是唯一遊戲中的引導（且是真實護理師會說的話）

> ⚠️ 這與 DESIGN-DIFFICULTY-SYSTEM.md 的「Standard Inline Guideline 觸發時機」衝突。
> **決議**：取消 Standard 遊戲中的 inline guideline feedback。護理師引導取代之。
> Guideline 呈現全部移到 Debrief。

### 原則 2：護理師是核心角色

護理師不是腳本化觸發訊息，是**有個性的 AI 角色**。

| 情境 | 護理師反應 |
|------|----------|
| 開 NS bolus 但 JVP 高 | 「學長，這個 patient JVP 蠻高的欸，確定要 bolus 嗎？」 |
| 猶豫太久 | 「學長，BP 又掉了，要不要先掛個什麼？」 |
| 開抗生素沒抽 culture | 「那 blood culture 要不要先抽？」 |
| 合理 order | 「好，我馬上掛」（簡潔） |
| 同時開衝突 order | 「學長，這兩個會不會衝突啊？」 |
| 劑量離譜 | 「學長，這個劑量我再確認一下喔...」 |

**設計要點**：
- 護理師 = 安全網 + 壓力源
- **不直接說「你錯了」**，用臨床常見方式暗示
- 有自己的節奏 — 不是即時回應，有時忙別床會晚回
- Standard 模式：引導更多（真實護理師對 junior 的反應）
- Pro 模式：只報數字（真實護理師對 senior 的反應）

**技術路線**：
- Phase 1-2：腳本化 trigger（現有 GuidanceEngine 的 7 種 trigger）
- Phase 3+：Claude API 驅動（scenario context + 玩家行為 → 動態生成）
- 這是比 Lite 模式更高 ROI 的 Claude 使用方式

> ⚠️ DESIGN-DIFFICULTY-SYSTEM.md 寫「❌ AI 對話護理師（Phase 3+ 才考慮）」
> **更新**：保持 Phase 3+ 時間表，但提升優先級。這是系統最大的體驗提升點。

### 原則 3：資訊碎片化 — 不一次給完

| 資訊 | 取得方式 | 等待時間 |
|------|---------|---------|
| Vital Signs | 自動顯示 | 即時 |
| 病史 | 問護理師/病人 | 即時 |
| 理學檢查 | 主動做 | 即時 |
| Lab | 開立後等 | 30-90 秒（模擬真實） |
| POCUS | 主動做 | 即時 |
| CXR | 要 order | 2-3 分鐘 |
| 學長意見 | 打電話 | 看報告品質決定回應 |

**Standard vs Pro**：
- Standard：lab 等待時間較短（15-45 秒），CXR 即時
- Pro：完整等待時間

### 原則 4：節奏感 — 安靜與緊張交替

- **安靜時刻**：等 lab、等藥物作用、觀察 vitals 趨勢
- **緊張時刻**：突然惡化、護理師急 call、vitals 跳警報
- 不是平的一條線 — 有起伏
- 靜默不代表安全（「學長…病人好像沒有呼吸了」）
- 遊戲設計上：每個 scenario 至少有 2 個節奏轉折

### 原則 5：Fog of War — 資料不完美

| 情境 | 設計 |
|------|------|
| Troponin 可能是舊值 | Lab 結果附「上次抽血時間」 |
| SpO2 probe 接觸不良 | 偶爾顯示 88% 然後恢復（假警報） |
| 護理師記錄 vs 自己量的 BP 不同 | 護理師報一個數字，monitor 顯示另一個 |
| Hemolysis 影響 K+ | Lab 結果附「hemolysis noted」|

**Standard vs Pro**：
- Standard：fog of war 較少（資料較乾淨）
- Pro：完整 fog of war

### 原則 6：Escalation 路徑 — 知道什麼時候 call 學長

| 玩家行為 | 學長（Claude）回應 | 教學意圖 |
|---------|-------------------|---------|
| 報告品質好 | 「你先處理，我等下來看」 | 鼓勵完整報告 |
| 報告太差 | 「你先講清楚，這病人怎樣？」 | 訓練溝通 |
| 太早 call | 「你先自己看一下再打來」 | 鼓勵獨立思考 |
| 太晚 call | 「怎麼現在才打！我馬上下來」 | 學會判斷時機 |

學長也用 Claude 驅動（Phase 3+）。Phase 1-2 用腳本。

## 各模式的體驗目標

### 玩完後的感受

**Lite**：「原來醫師在 ICU 面對的壓力是這樣。要在不確定的情況下做決定好難。」

**Standard**：「原來 ICU 被 call 是這種感覺。我下次去值班前要先想清楚怎麼分。」

**Pro**：「護理師那句『JVP 蠻高的欸』我應該要注意到的。」

### 非醫護玩家考量

| 層面 | 適配方式 |
|------|---------|
| 術語 | 關鍵醫學術語附簡短中文（hover tooltip）|
| 選項 | 每個醫囑附一行白話描述（context，不是教學）|
| 護理師 | 更口語化，用白話解釋狀況 |
| Debrief | 增加「背景知識」區塊 |
| 難度 | Lite 可選「學習模式」— 不 Game Over |

**不做的事**：
- 不把遊戲變教科書 — 核心是體驗
- 不降低醫學準確性 — 簡化解釋但不扭曲事實
- 不在遊戲中教基礎知識
